## Stack
- **Backend**: C++ (lista encadeada manual, POO, persistência em memória)
- **Parser/Extração**: Python (extração de texto dos boletins PDF → CSV)
- **Frontend**: Next.js (dashboard com visualizações)
- **Formato intermediário**: CSV (Python gera, C++ consome) + JSON (C++ gera, Next.js consome)

## Fluxo de Dados
```
PDFs da prefeitura
      ↓
  Python (Pedro) — extrai dados → CSV em data/processed/
      ↓

      ↓
  Next.js (Gabriel + Alan) — lê resultado.json → exibe no dashboard
```

## Arquitetura C++

### Classes
- `Localidade` (abstrata) — classe base com m_nome e calcularRisco() virtual puro
  - `RegiaoUrbana` — Norte, Sul, Leste, Oeste, Central
  - `DistritoRural` — Rural
- `BoletimSemanal` — dados de uma região em uma semana epidemiológica
- `No` — nó da lista encadeada (aponta para Localidade e próximo nó)
- `ListaEncadeada` — lista principal de regiões
- `LeitorCSV` — lê CSV e popula a lista
- `AnaliseSituacional` — percorre a lista e gera relatórios

### Estrutura de dados em memória
```
ListaEncadeada
  └── Nó → RegiaoUrbana("Norte")
               └── sublista: [sem 01] → [sem 02] → [sem 03] → null
  └── Nó → RegiaoUrbana("Sul")
               └── sublista: [sem 01] → [sem 02] → ...
  └── Nó → DistritoRural("Rural")
               └── sublista: [sem 01] → [sem 02] → ...
```

### calcularRisco()
- RegiaoUrbana: `(confirmados / notificados) × 100`
- DistritoRural: `(confirmados / notificados) × 150` (fator de subnotificação)

## Regras do Professor (inegociáveis)

### POO
- Mínimo 4 classes interconectadas
- **Encapsulamento**: todos os atributos PRIVADOS, acesso via getters/setters
- **Herança**: Localidade → RegiaoUrbana / DistritoRural
- **Polimorfismo**: calcularRisco() diferente por tipo de localidade

### Estrutura de Dados
- Lista encadeada implementada **MANUALMENTE** — PROIBIDO usar std::list, std::vector ou qualquer container da STL para a estrutura principal
- Operações obrigatórias:
  - `inserirOrdenado()` — por índice de risco (maior risco primeiro)
  - `remover()` — excluir localidade por nome
  - `buscar()` — localizar localidade por nome
- Sublista de BoletimSemanal dentro de cada Localidade, em ordem cronológica

### Relatório Situacional (AnaliseSituacional)
- Região com maior risco (calcularRisco() do boletim mais recente)
- Taxa de crescimento de confirmados entre último e penúltimo boletim
- Filtro por localidade com histórico cronológico

## Convenções de Código

### C++
- Compilar com `g++ -std=c++17 -Wall -Wextra`
- Nomes de classe em PascalCase: `ListaEncadeada`, `BoletimSemanal`
- Nomes de métodos em camelCase: `inserirOrdenado()`, `buscar()`
- Atributos privados com prefixo `m_`: `m_nome`, `m_notificados`
- Um .h e um .cpp por classe
- Sem `using namespace std` no header

### Python
- Python 3.10+
- Nomes em snake_case
- Saída sempre no formato do contrato CSV

### Git
- Branch `main` protegida
- Features em branch: `feat/lista-encadeada`, `feat/parser-csv`, `feat/wireframes`
- Commits em português, prefixados: `feat:`, `fix:`, `docs:`, `refactor:`
- PRs com descrição mínima do que foi feito
